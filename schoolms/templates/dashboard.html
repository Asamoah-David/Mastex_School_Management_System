How schools, parents & staff get access (SaaS)

1. Schools
You add a school in Django Admin → Schools. Each school gets a unique subdomain and Stripe/Flutterwave billing. The school admin is created and invited.
2. Staff (teachers, school admins)
Created under Accounts → Users with school and role set. They log in at the same URL and see only their school's data.
3. Parents
When a Student is linked to a parent (User with role=parent), the parent logs in and sees their children on the Parent Dashboard. You create parent users and link them to students.
4. Students
Each student has a User (role=student) linked via Student.user. Students can log in to view results and pay fees.

Next step: Add a School, then create Users (staff/parents/students) and link them. Use Django Admin for full CRUD. For a public school signup flow, you can add a registration page later.
